# Git を使った論文執筆ワークフローの設計

## 背景
技術系論文をgit管理しながら執筆しています。現在は査読フェーズです。

## 質問
論文執筆におけるgitワークフローのベストプラクティスを教えてください。

## 知りたいこと
- **コミット戦略**：どのタイミング・粒度でコミットすべきか
- **ブランチ戦略**：いつブランチを分けるべきか（査読対応、大幅改訂など）
- **タグ付け**：どのタイミングでタグを打つべきか（投稿版、査読返却版など）
- **査読対応フロー**：査読コメントへの対応をどう追跡するか
- **バージョン管理**：v1投稿 → 査読 → v2改訂 → 再査読 の流れ

## 現在のリポジトリ構造
```
paper-driven-dev/
├── paper/           # 論文本体
├── docs/            # 補足ドキュメント
├── .internal/       # 作業メモ・下書き
└── ...
```

## 成果物
`.internal/notes/git-workflow-model.md` に以下を出力：
- 推奨ワークフローモデル
- 各フェーズでのgit操作例
- 命名規約（ブランチ名、タグ名）
