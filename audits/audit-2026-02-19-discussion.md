## Round 1: 論点提示

### Agent A（Claude 査読者）

本監査で最も重大な発見は、**循環論法**と**交絡因子の未分離**の2点である。これらは Codex 監査では検出されなかった独自発見であり、論文の中心的主張を根底から揺るがす。

**1. 循環論法（P0）**

論文の核心的主張は「論文形式で聞くだけで、4つの行動が指示しなくても自然に出現する」（README.md:5, paper.md:11-16）である。しかし SKILL.md は「at least 3 approaches を列挙せよ」（L57）、「testable properties を定義せよ」（L68）、「Given/When/Then 形式で書け」（L69）と**明示的に指示**している。出力にこれらが含まれるのは当然であり、「自然に出現した」とは言えない。

**2. 交絡因子の未分離（P1）**

Before は1行の汎用プロンプト、After は7セクション・詳細ガイドライン付きテンプレートである。この比較では「論文形式の効果」と「構造化テンプレートの効果」と「各セクションの具体的指示の効果」が分離できない。同じ §1-§7 構造を「設計文書」と呼んで使っても同じ品質が出るなら、効いているのは「論文形式」ではなく「構造化テンプレート」である。

これらは独立した問題ではなく、**同根**である。本プロジェクトの実際の貢献は「構造化テンプレートによる品質の安定化」だが、主張は「論文形式という魔法のフレーミング」として提示されている。このずれが他の多くの指摘の根本原因である。

### Agent B（Codex 査読者）

我々の監査は6件の指摘を行い、最優先は以下の2件であった。

**1. 実験条件の文書間不整合（P1）**

paper.md は「Claude で N=1」、comparison-data.md は「o3 で N=2」、article-draft.md は「N=1（モデル不明）」。同一リポジトリ内に異なる実験条件のデータが無区別で並置されており、読者は単一の実験セットと誤解する。

**2. 主張強度と証拠強度の不釣り合い（P1）**

README.md の「劇的に上がる」、guide.md の「自然に出現」という断定的表現は、N=1-2・著者評価・単一モデルという証拠の強度と釣り合っていない。

Agent A が指摘する循環論法については、興味深い観点だが、**過度に厳格な読み**ではないかという疑問がある。論文の paper.md:11 は "four behaviors emerge without explicit instruction" と書いているが、これは「テンプレートなしの状態でも」という文脈で読むのが自然ではないか。テンプレートはツールとしての実装であり、論文の観察はテンプレート開発の**動機**を説明している。つまり、まず「論文を書け」で品質向上が観察され、次にそれをテンプレートとして安定化した、という時系列がある。

### Agent C（論文執筆者）

両査読者に感謝する。まず、共通して指摘された点について認める。

**認める点:**
- 実験条件の不整合は確かに問題であり、修正する（Agent B 指摘1）
- 「劇的に」等の断定的表現が証拠強度に見合っていないのは認める（Agent A 指摘6, Agent B 指摘2）
- Before プロンプトが意図的ではないにせよ結果的に弱いのは事実（Agent A 指摘3）

**反論する点:**

Agent A の循環論法の指摘については、Agent B が正しく指摘したように**時系列の混同**がある。PDD の発見プロセスは以下の通りだった:

1. まず「この問題について学術論文の形式で書いてください」と一言プロンプトで試した
2. テンプレートなしで4つの行動が出現した
3. これを安定的に引き出すためにテンプレート（SKILL.md）を作成した

テンプレートは「自然に出現する現象」の**後付けの安定化**であり、循環論法ではない。問題は、この時系列をリポジトリ内で明確に説明していない点にある。

交絡因子については、**部分的に認める**。確かに現在の比較では「論文形式」と「構造化テンプレート」の効果を分離できていない。ただし、paper.md §5.1 で3つの仮説を提示し、§5.3 で因果検証が将来課題であると明言している。主張は「因果関係の証明」ではなく「観察された現象の報告と仮説提示」である。

## Round 2: 反論・深掘り

### Agent A（Claude 査読者）

Agent C の「時系列の混同」という反論を検討する。

確かに、テンプレート開発の動機が「テンプレートなしで観察された現象」にあったという説明は論理的に成立しうる。**しかし、問題はそこではない。** 問題は以下の2点である:

**第一に、エビデンスの不在。** 「テンプレートなしでも出現した」という主張を裏付けるエビデンスがリポジトリ内に一切存在しない。提示されている Before/After サンプルはすべてテンプレート経由の出力である。読者が検証できない主張は、学術論文としては単なるアネクドートに過ぎない。

**第二に、文書の記述が時系列を反映していない。** README.md:5 は「指示しなくても自然に出現する**現象を活用した**開発手法」と書いているが、SKILL.md は明示的に指示している。読者は SKILL.md を使って再現を試み、「テンプレートが指示しているから出現するのでは？」と疑問を持つだろう。「テンプレートなしでもまず出現し、テンプレートでそれを安定化した」という時系列は読者に伝わらない。

Agent B の「過度に厳格な読み」という指摘に対しては、**学術論文として提出する文書に対して学術的厳格さで読むのは当然**であると反論する。ブログ記事なら許容される曖昧さも、論文体裁の文書では瑕疵となる。

### Agent B（Codex 査読者）

Agent A の「エビデンスの不在」という指摘は正当である。これは我々の監査で見落とした点であり、認める。

しかし、Agent A の再反論には**論点のすり替え**がある。Agent A は当初「循環論法」と呼んだが、今の議論は「エビデンスの不在」と「読者への伝達の不備」に移行している。これらは異なる問題である:

- **循環論法**: テンプレートが要求するものを「自然に出現する」と主張 → 論理構造の欠陥
- **エビデンスの不在**: テンプレートなしの出力が提示されていない → 検証可能性の欠陥
- **伝達の不備**: 時系列が読者に伝わらない → 文書品質の欠陥

Agent C の反論が正しいなら、これは「循環論法」ではなく**「不十分な証拠提示」**である。重大度は変わらないかもしれないが、診断名は正確であるべきだ。

交絡因子について補足する。我々の監査では交絡因子を独立した指摘として取り上げなかったが、それは意図的な判断ではなく、**論証構造よりも表面的整合性に焦点を当てた結果の見落とし**であった。Agent A のこの発見は妥当であり、我々の監査の限界を示している。

### Agent C（論文執筆者）

Agent A の2点の反論は正当であり、受け入れる。

**エビデンスの不在について:** 「テンプレートなしで出現した」という開発動機は、確かに検証不可能なアネクドートのままである。修正方法として以下を提案する:

- **案1**: テンプレートなしの「論文形式で書いてください」一言プロンプトの出力を新規に取得し、Before/After の中間条件として追加する（Agent A Phase 2 推奨アクション5と一致）
- **案2**: 主張を修正し、「テンプレートなしでも出現する」ではなく「テンプレートにより安定的に引き出せる」と書き換える

**案2を即時実行し、案1を次版で実行**するのが現実的と考える。

Agent B の「循環論法」vs「不十分な証拠提示」の診断名の区別は重要で、正確な指摘である。私の時系列説明が正しいなら循環論法ではないが、その時系列を証拠で示せていない以上、読者にとっては循環論法と区別がつかない。実質的な修正内容は同じだが、問題の本質を正しく理解することで、より適切な修正ができる。

## Round 3: 交絡因子と比較の公平性

### Agent A（Claude 査読者）

Agent B の診断名の修正を受け入れる。「循環論法」は撤回し、**「主張とエビデンスの不対応：テンプレートなしの中間条件が欠如」**と再定義する。重大度は P0 のまま変更しない。

次に、交絡因子と Before プロンプトの弱さについて議論を深めたい。

Agent C は「因果関係の証明」ではなく「観察の報告と仮説提示」と主張した。これについて2つの問題がある。

**第一に、paper.md の記述は「観察の報告」を超えている。** paper.md:196 の結論は "Paper-Driven Development demonstrates that instructing LLMs to produce design analysis in academic paper format **systematically improves** output quality" と書いている。"demonstrates" と "systematically improves" は因果的主張の言語である。「観察した」（"we observed"）とは異なる。

**第二に、comparison-data.md の比較構造は因果推論を誘導する。** Before/After の対比表は、差分の原因が「PDD」にあるという因果的解釈を暗黙に促す。これは「不公平な比較を注意深く読めば分かる」レベルの問題ではなく、文書の構造自体が因果推論を誘導している。

修正提案: paper.md の結論を "we observed that ... is associated with improved output quality in our case study" に変更すること。

### Agent B（Codex 査読者）

Agent A の指摘に同意する。"demonstrates" は実験科学の文脈では「実証した」を意味し、N=1-2 のケーススタディには強すぎる。

Before プロンプトの弱さについて補足する。我々の監査ではこれを独立した指摘として取り上げなかったが、comparison-data.md:88 の "The conventional prompt was not optimized" という注記の存在を確認していた。

ここで Agent C に問いたい: **Before プロンプトを最適化しなかったのは意図的な方法論上の選択か、それとも見落としか？** もし意図的であれば、その理由を文書化すべきだ（例:「実務でよくある平均的なプロンプトとの比較を意図した」）。見落としであれば、追加実験が必要だ。

### Agent C（論文執筆者）

**"demonstrates" の修正:** Agent A の指摘を受け入れる。"demonstrates" → "suggests" または "we observed that ... is associated with" に変更する。

**Before プロンプトについて:** Agent B の質問に対し、正直に答える。**意図的な選択ではなく、当初は比較の公平性を深く考慮していなかった。** comparison-data.md:88 の注記は監査を見越して後から追加したものではなく、データ作成時に「これは最適化されていないな」と気づいて書いたメモである。

しかし、ここで反論もある。**Before プロンプトを「最適化しなかった」ことには、一定の合理性がある。** PDD の実用上の比較対象は「開発者が普段使っている程度のプロンプト」であり、CoT やペルソナプロンプティングを駆使した最適化プロンプトではない。学術的に厳密な比較と、実用上の有用性を示す比較は異なる。

ただし、この主張は両立可能だ:
- 実用上の比較: 「普段のプロンプト vs PDD」 → 現在のサンプルで十分
- 学術的な比較: 「最適化されたプロンプト vs PDD」 → 追加実験が必要

問題は、**論文体裁の文書で実用上の比較のみを提示していること**である。論文として書くなら学術的な比較基準が求められる。article-draft.md（ブログ記事）ではこの比較でも許容されるだろうが、paper.md では不十分である。

## Round 4: 実験条件の不整合と修正の方向性

### Agent B（Codex 査読者）

Agent C の「実用上の比較 vs 学術的比較」の区別は明確で有用である。この区別に基づき、修正方針を整理したい。

実験条件の不整合について、Agent A の Round 2 深掘りで興味深い転換案が示されていた:「不整合を多モデル再現性の予備的証拠として積極的に位置づけ直す」。我々もこの方向性に同意する。具体的には:

```
Experiment A (paper.md): Claude, N=1, 2026-02
Experiment B (comparison-data.md): o3, N=2, 2026-02
```

この2つを別実験として明示し、「異なるモデルで類似の傾向が観察された」と記述すれば、弱点が予備的な強みに変わる。ただし、以下の条件が前提:

1. 各文書冒頭にメタ情報（Model, N, Date, Evaluator）を明記
2. 「同一実験ではない」ことを明示
3. 統計的な一般化は主張しない

## Round 5: 残論点の整理と合意形成

### Agent A（Claude 査読者）

Agent C の修正提案（メタ情報のカテゴリ1移動）に同意する。

残る論点として、**paper.md:198 の過剰な一般化**がある。「paper format does not help LLMs find better answers」は N=2 のケーススタディから導出された一般的主張であり、Agent C は我々の P2 判定をどう受け止めるか。

### Agent B（Codex 査読者）

我々の監査ではこの一般化を検出できなかったが、Agent A の指摘は妥当と考える。N=2 でいずれも「明確な最適解がある問題」だったことを考えると、「設計空間が広い問題では異なる結論に至る可能性がある」というのは合理的な懸念である。

ただし、**この主張は PDD に対して好意的でない方向の一般化**である点に注目したい。PDD が「答えの質を向上させない」と述べることは、PDD の主張を控えめにする効果がある。過剰な一般化ではあるが、読者を誤導する方向は「PDD を過大評価させる」ではなく「PDD を過小評価させる」である。修正の優先度は P2 で妥当。

### Agent C（論文執筆者）

Agent A の指摘と Agent B の補足を受け入れる。修正は容易であり、以下のように限定句を追加する:

```diff
- paper format does not help LLMs find better answers
+ in our case studies (both involving problems with relatively clear optimal solutions),
+ paper format did not lead to different design conclusions
```

これにより、一般化を避けつつ、「答えの質ではなく正当化の質が向上する」という PDD のポジショニングを維持できる。

### Agent A（Claude 査読者）

Agent C の修正案で十分である。

最後に、全体評価について。我々は B、Agent B は B- と評価した。この差は主に「論証構造の重大な欠陥を、内容の実用的価値で相殺できるか」の判断基準の違いに起因する。修正後の評価見込みについてはどう考えるか。

### Agent B（Codex 査読者）

B- とした理由は「実証記述の統制が弱い」であった。カテゴリ1の修正（主張の緩和 + メタ情報追加）が完了すれば、「主張と証拠が整合した状態」になるため、**B+ まで向上可能**と考える。カテゴリ2（追加実験）が完了すれば A- も射程に入る。

ただし、これは**文書の品質評価**であり、**研究の品質評価**ではない。研究としての評価は、追加実験なしには本質的に変わらない。

### Agent C（論文執筆者）

両査読者の評価基準の違いを理解した。Agent B の「文書の品質と研究の品質は別」という区別は重要である。

本プロジェクトの現実的な位置づけを明確にする:
- **研究**: 予備的な観察と仮説提示（pilot study レベル）
- **文書**: その観察を正確に伝える報告書
- **ツール**: 実用的な Claude Code プラグイン

3つの側面を混同して「論文」と称していたことが、多くの問題の根本にある。

### 3. 議論から得られた追加の洞察

1. **2レイヤー仮説**: PDD には「論文ラベルの単独効果」と「テンプレートによる安定化効果」の2つのレイヤーがあり、現在のエビデンスは後者のみを示している。前者のエビデンスが追加されれば、主張の独自性が大幅に強化される。

2. **文書と研究とツールの分離**: 「論文」「ブログ記事」「ツールドキュメント」で許容される主張の強度が異なる。各文書の性質に応じた主張のキャリブレーションが必要。

3. **弱点の転換**: 実験条件の不整合（Claude vs o3）は、適切に記述すれば「多モデル再現性の予備的証拠」に転換できる。

4. **Agent B 自己評価**: Codex 監査は論証構造の分析が不十分であった。表面的整合性のチェックに偏り、循環論法・交絡因子・Before の弱さ・過剰な一般化を見逃した。

5. **監査における診断の精度**: 「循環論法」と「エビデンスの不足」は異なる問題であり、診断名の正確さは修正の方向性に影響する。批判的監査においても、指摘の正確な言語化は重要である。
