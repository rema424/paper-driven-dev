# 議論: フルペーパー化ロードマップ（2026-02-20）

## 問いの本質（Codex 定義）

> ソロ研究者の現実的制約下で、探索的ケーススタディをフルペーパー査読の必須水準（内的妥当性・再現性・一般化可能性）へ引き上げる段階的ロードマップをどう設計するか。

## 初期立場

### CC
3フェーズ構成（内的妥当性 → 一般化 → 差別化）。マルチモデルは「強く推奨」、下流アウトカムは「望ましい」。

### Codex
4フェーズ構成（Phase 0 計画固定を追加）。マルチモデルは「必須」、下流アウトカムは「強く推奨」。

## 相違点と解決

1. **マルチモデル**: 主張範囲に依存。「外的妥当性の1軸（マルチモデル or 追加ドメインのどちらか）」を準必須とすることで合意。
2. **Phase 0**: CC が欠落を認め、追加に同意。低コスト・高防御力。
3. **フルペーパー最低ライン**: Phase 1 + Phase 2-lite（外的妥当性1軸）が安全ライン。

## 最終合意

### 分類

| 分類 | 項目 |
|------|------|
| **必須** | Phase 0 計画固定、D 条件、N≥5、第三者盲検（co-primary 2指標） |
| **準必須** | 外的妥当性1軸（追加1モデル or 追加1ドメイン） |
| **強く推奨** | 外的妥当性2軸の両達成、下流アウトカム1指標以上 |
| **望ましい** | 強化版プロンプト比較、追加ドメイン拡張、コスト効率分析 |

### フェーズ構成

**Phase 0（計画固定）**
- タスク: 仮説、co-primary 2指標、解析手順、除外基準、盲検手順を事前固定
- 成果物: タイムスタンプ付きプロトコル（Git tag/OSF）、評価ルーブリック v2、実験実行テンプレート
- 完了条件: 変更履歴付きで固定完了、逸脱理由を記録できる状態
- Go/No-Go → Phase 1: プロトコル固定完了

**Phase 1（内的妥当性の確立）**
- タスク: GPT-5.2 で A/B/C/D を N≥5 で実施、第三者盲検（2名以上）で co-primary 採点、ルーブリック校正
- 成果物: 条件別データ一式、盲検採点表、Cohen's κ、実験ログ
- 完了条件: D 条件を含む比較成立、κ≥0.60
- Go/No-Go → Phase 2: κ 基準達成。未達ならルーブリック再校正して再試行

**Phase 2-lite（外的妥当性の安全ライン）**
- タスク: 追加1モデル or 追加1ドメインを実施（どちらか必達）
- 成果物: 外的妥当性の追加検証結果、主張範囲の明確化文
- 完了条件: 少なくとも1軸で効果方向の再現確認
- Go/No-Go → 投稿: 再現確認。未達なら主張を限定するか追加実験

**Phase 3（差別化強化・任意）**
- タスク: 下流アウトカム、強化版プロンプト比較、外的妥当性第2軸
- 成果物: 実務的有用性の補強結果
- 完了条件: 差別化材料が揃う

### 実行上の注意点（ソロ研究者向け）

- 第三者評価: 2指標限定・60-90分・評価パッケージ事前作成で依頼
- 評価者は早期に2名確保、校正ラウンド後に本番盲検
- 実験実行と集計を自動化、手作業は採点運営に集中
- 各フェーズ末で中止基準を明示、過剰拡張を防ぐ

### 想定投稿先

- SE系（最有力）: 実験設計・比較条件・再現性を主軸
- HCI系: 人間評価設計やワークフロー有効性を強める場合
- AI系: LLM評価方法論として位置づける場合
