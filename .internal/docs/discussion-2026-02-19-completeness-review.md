# 議論: 改訂後論文の完成度評価と投稿前残課題（2026-02-19）

## 発端

3条件比較+Bバリアント実験の議論合意に基づく論文全面改訂（8コミット）の完了後、投稿に向けた完成度をCC×Codexで評価した。

## 問いの本質（Codex 定義）

> 改訂後の論文と関連文書が想定投稿先で受理可能な完成度に達しているかを客観的基準で点検し、投稿前に解消すべき残課題を優先順位付きで特定すること。

## 初期立場

### CC の初期立場

投稿可能レベルには近いが構造的・内容的課題が残る。

- 強み: 主張と証拠の整合性、実験デザインの透明性、限界の誠実な開示、Tier 1/Tier 2 の実務的価値
- P0: S3.2 配置問題、constraints disclosed の二重計上（A/B で 1.5 なのに「ゼロ」と記述）
- P1: S2 Background の接続不足、S6 Related Work の薄さ、formal invariants 未報告
- P2: Abstract の長さ、ドメイン多様性の限界、B3 evaluation metrics の議論不足

### Codex の初期立場

Workshop は適切、短報は条件付き、フルペーパーは未達。

- 主張と証拠の整合性は探索的水準では成立するが、N=2・著者評価・盲検なしで因果・一般化の裏付けが不足
- P0: D条件追加、盲検+複数評価者+ルーブリック導入、再現可能性パッケージの整備
- P1: N・ドメインの拡張、下流アウトカムの追加

## 批評と論点整理（Codex 主導）

### 合意点

- 過大主張を抑えており探索的研究としての誠実性は高い
- A/B/C + B1/B2/B3 の設計と Tier 1/Tier 2 指針は実務的価値がある
- フルペーパー水準には届かず、まずは workshop（条件付きで短報）

### 相違点

- Codex: 推論妥当性（因果・一般化）を最優先
- CC: 原稿内整合性・構成品質を最優先
- Codex: 方法論的補強なしでは評価が頭打ち（より慎重）
- CC: 投稿可能に近い（やや前向き）

### 批評の要点

- CC の弱点: N=2・著者評価という中核的妥当性リスクへの処方が弱い
- Codex の弱点: 原稿固有の整形不良（constraints の二重基準、formal invariants 未報告）を十分に拾えていない
- 両立場は対立ではなく補完関係: CC は「原稿の整形不良」を、Codex は「証拠の強度不足」を捉えている

## ラリーでの修正

### CC の修正

- 推論妥当性の優先度を引き上げ、S3.2 配置問題は P1 に降格
- constraints disclosed の記述矛盾は P0 として維持
- 再現可能性パッケージの必要性に同意
- formal invariants は「全条件で差なし」と1文報告する案を提案
- 第三者確保が困難な場合、self-blinded 再採点を提案

### Codex の修正

- formal invariants の1文報告に同意
- self-blinded 再採点を workshop レベルなら条件付きで許容（独立検証とは表現せず「補助的ロバストネス確認」として位置づけ）

### 追加実験の境界

- **実験なしで対応可能**: 記述矛盾修正、主張限定化、formal invariants 報告、再現可能性パッケージ整備、既存出力の self-blinded 再採点
- **追加実験が必要**: D条件新設、N≥5、第三者盲検評価、マルチモデル・マルチドメイン検証、下流アウトカム測定

### constraints disclosed の修正方針

現状の問題: Abstract、S4.2、S8 で「co-primary indicators remained at zero across all framing variants」と書いているが、constraints disclosed は A/B でも 1.5 でゼロではない。conflicting requirements と testable properties はゼロ→非ゼロだが、constraints は 1.5→4.0 の増加。

修正方針: 3指標をまとめて「ゼロ」と表現するのではなく、指標別に記述する。
- conflicting requirements: 0 → 2.5（A/B でゼロ、C でのみ出現）
- testable properties: 0 → 6.5（A/B でゼロ、C でのみ出現）
- constraints disclosed: 1.5 → 4.0（A/B でも存在するが、C で大幅に増加）

### self-blinded 再採点の手順

1. 全出力ファイル（12件: 6条件 × 2ケーススタディ）のファイル名から条件ラベルを除去
2. ランダム順に並べ替え
3. 固定ルーブリック（co-primary 3指標 + secondary 指標の定義と計数基準）に基づき再採点
4. 元の採点結果と照合し、一致率を報告
5. 手順と結果を S5.5 または付録に記載
6. 「著者による self-blinded 再採点であり、独立した第三者評価ではない」と明記
