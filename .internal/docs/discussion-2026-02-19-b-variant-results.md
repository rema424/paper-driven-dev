## 初期立場

### CC の初期立場

LLM の設計分析出力は「フレーミング軸」と「テンプレート軸」の2つの独立した改善軸を持つ。論文は「2軸独立モデル」を中心に構成すべき。

- フレーミング効果: トートロジー問題なし、ゼロコスト
- テンプレート効果: 劇的だがトートロジー問題あり
- 実務的価値: Tier 1（フレーミング）/ Tier 2（テンプレート）

### Codex の初期立場

「一言で体裁は変わるが、価値を生む設計品質は構造でしか安定して変わらない」。

- 最も強い証拠: B1/B2/B3 で conflicting requirements=0、testable properties=0 が維持
- 差分は「正解到達」ではなく「仕様化可能性」
- フレーミング効果は体裁改善が中心

## 批評と論点整理（Codex 主導）

### 合意点

- B1→B3 で学術的体裁と一部分析深度は上がる
- conflicting requirements と testable properties は C でのみ一貫して増える
- N=2・単一モデルで強い一般化は不可

### 相違点

- CC: 2軸独立モデルを主貢献に / Codex: 構造化効果が主、2軸は説明枠
- CC: フレーミング効果も発見として重視 / Codex: フレーミングは体裁中心
- CC: LLM学習データ活性化仮説を含む / Codex: 仮説は魅力的だが検証不能

### 批評の要点

- CC の弱点: 「独立性」は示唆レベルで直交設計で未検証。フレーミング効果は modest。
- Codex の弱点: 「構造でしか」は強すぎ。B3 の実質改善（approaches 2.0→3.5、評価計画セクション）を過小評価。
- 共通: トートロジー問題の扱い

## CC の修正

- 主貢献の階層化に同意（テンプレート効果が主、2軸モデルは説明枠）
- フレーミング効果を「探索幅と形式性の改善」に限定
- トートロジーの「外在化」リフレーミングを採用
- B3 の評価指標セクションを「フレーミング効果の天井」として位置づけ
- 「一言変えるだけで変わる」をフック→「しかし不十分」→テンプレートへの導入構造

## Codex の最終修正

- 「体裁中心」をやや緩和（B3 の実質改善を認める）
- S1 フック: 「品質が変わる」→「出力特性（探索幅・形式性）が変わる」に限定
- 「安定的に促進」→「本データ内で一貫して観測された」に弱化
- 3指標は co-primary として個別報告

## 最終合意

### 論文の新しい中心主張（1文）

GPT-5.2・N=2 の探索的ケーススタディにおいて、指示フレーミングの変更は出力特性（探索幅・形式性）の改善と関連した一方、co-primary 指標である conflicting requirements・testable properties・constraints disclosed の出現は、本データ内では PDD テンプレート使用とより強く関連して観測された。

### 各セクションの改訂方針

| セクション | 方針 |
|-----------|------|
| **S1** | フックを「一言で出力特性は変わる」に設定し、直後に「ただし検証可能性は不十分」を明示して主貢献へ接続 |
| **S3** | 評価枠組みを明確化。2軸モデルは説明枠として提示し、co-primary 3指標を主要アウトカムとして定義。体裁品質は副次アウトカム |
| **S4** | A/B/C と B1/B2/B3 を統合提示。主分析は co-primary 3指標、B3 の実質改善は副次分析として定量・定性で報告 |
| **S5** | 「フレーミング効果の天井」を記述。探索幅・形式性は改善するが co-primary は不変、という質的差を議論 |
| **S7** | Future Work に D条件、多モデル反復、盲検評価、行あたり正規化、下流成果測定を明記 |
| **S8** | 結論は限定主張に徹し、実務への Tier 1 / Tier 2 適用指針を提示 |

### 主張の上限（言い切ってはいけないこと）

- 「テンプレートが因果的に品質を改善した」
- 「フレーミングが研究者ペルソナを活性化した」と断定
- 「有意」「実証済み」「一般化可能」とする表現
- 「品質向上」を包括的に断定する表現

### 正直に認めるべき限界

- N=2、単一モデル（GPT-5.2）、single-run
- 著者評価によるバイアス可能性
- 指標とテンプレート要求の近接によるトートロジー懸念（「外在化」として記述）
- 出力量増加と指標増加の交絡、行あたり正規化未実施
- D条件（論文形式なしのチェックリスト）の欠落
- CS2-B3 復元データの不確実性

### 表現ルール

- 主動詞は "was associated with" で統一
- テンプレート側: 「本データ内で一貫して観測された」
- フレーミング側: 「改善に関連した」
- 「improves / demonstrates / proves / significant」は不使用

### 実務的含意（Tier 1 / Tier 2）

- **Tier 1（ゼロコスト）**: フレーミングを変更（「分析して」→「論文を書いて」）し、探索幅・形式性を引き上げる
- **Tier 2（テンプレートコスト）**: 重要設計判断ではテンプレートを使い、検証可能性に直結する情報の外在化を狙う

### 前回議論からの変更点

- 主張に B バリアント知見を正式統合し、「フック→限界→解」の流れを明示
- 主要アウトカムを co-primary 3指標として固定し、approaches は副次扱いへ整理
- 「安定的に促進」を「本データ内で一貫して観測」に弱化
- S3 に評価枠組み（2軸は説明枠、主分析は co-primary）を追加明確化
- S5 で「フレーミング効果の天井」を明示し、B3 の位置づけを統一

## 補足: 「論文を書いて」の論文内での位置づけ

ユーザーから「つまり『論文を書いて』は着想のきっかけという触れ方になるのか？」と確認があった。

合意における位置づけは「着想のきっかけ以上、主貢献未満」の3層構造:

1. **フック（S1）**: 「一言変えるだけで出力特性が変わる」— 読者の関心を引く入口
2. **副次的な実証的発見（S4-S5）**: B1/B2/B3 の定量データに基づく「フレーミング効果の天井」の発見。単なるきっかけではなく、データで裏付けられた副次的知見
3. **主貢献を強化する反面教師（S5）**: 「フレーミングだけでは co-primary 指標は動かない」という事実が、テンプレートの価値を逆説的に証明する

ユーザーが当初期待していた「一言変えるだけで質が変わる、それが主張の柱」という方向性からは後退している。データが co-primary 指標の改善を支持しなかったため。ただし、「フレーミング効果の天井」という概念自体は論文のストーリーに不可欠な要素として残る。
